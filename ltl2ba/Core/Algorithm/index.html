Data type definitions

type red_states = {
all : Core.Automata.StateSet.t;(*

Set of all reachable reduced states using only unmarked edges.

*)
marked_by : Core.Automata.StateSet.t Core.Automata.FormulaMap.t;(*

Maps a formula α to the set of only reachable reduced states by using edges marked with α.

*)
}

Intermediate representation allowing to store reduced states computed by the red function.

Each formula used as a key of the map marked_by corresponds to an acceptance condition. More precisely, Redα = {(marked_by \ marked_byα) U all}.

val empty_red_states : red_states

empty_red_states returns the red_states with an empty all and an empty marked_by.

Functions

Main logic functions

val red_states_union : red_states -> Core.Automata.StateSet.t

red_states_union red_states Returns the union of all StateSet.t in red_states.

val next : Automata.state -> Automata.state

next state returns {α | Xα ∈ state}

val sigma : Automata.state -> Core.Automata.FormulaSet.t

sigma state returns the intersection of all p and ¬p in Z, p ∈ AP.

val is_reduced : Automata.state -> bool

is_reduced state returns true if the state is reduced.

val is_maximal : Ltl.formula -> Automata.state -> bool

is_maximal phi state returns true if the formula phi is maximal in state, i.e. phi is not a sub-formula of any other such formula of state.

val red : Automata.state -> red_states

red state returns the red_states corresponding the reduction of the state state.

Where:

  • all is equal to {Z reduced | state ⟶* Z}
  • and the value of marked_by associated to the key α equals to {Z reduced | state ⟶* without using an edge marked by α}
